TCP Data Protocol

Each Pico connects to 0.0.0.0:9991 and sends plain-text key-value pairs (one per line). The server parses every line and updates the corresponding global variable.

Async Task Architecture

The server runs 7 concurrent asyncio tasks inside a single Python process.

data_recv()
⏱ Continuous (0.2 s delay)
TCP socket listener. Accepts incoming connections from any Pico on port 9991, decodes the payload, parses each line, and updates global sensor variables. Increments a per-sensor sequence counter on every successful parse.
data_send()
⏱ Every 120 seconds
Sends current sensor values to ThingSpeak via HTTPS GET for all four channels (3 greenhouse channels + 1 RAM channel). On failure, posts an error alert to the Discord error channel.
transfer_discord()
⏱ Every 7200 seconds (2 hr)
Posts a RAM usage summary for all three Pico devices to the main Discord status channel. Allows remote monitoring of device health without SSH access.
backup_data()
⏱ Every 600 seconds (10 min)
Appends the current sensor snapshot (timestamp + all 12 readings) to an in-memory list and writes the entire list to backup_data_server.csv using pandas.
control_RPI()
⏱ Every 3600 seconds (1 hr)
Pings Pico 1 at 192.168.1.59 using the system ping command. Prints ONLINE / OFFLINE to the console. Helps detect network outages early.
control_RPI2() & control_RPI3()
⏱ Every 3600 seconds (1 hr)
Same as control_RPI() but for Pico 2 and Pico 3 at 192.168.1.118. Each runs as an independent task so one slow ping doesn't block the others.
Operational Timeline
T + 0 s — Startup
Server binds TCP :9991, initialises all globals to 0, posts "Program was started" to Discord main channel.
T + 120 s — First ThingSpeak push
All four ThingSpeak channels updated with the first batch of sensor data. Heartbeat "program is running!" printed to console.
T + 600 s — First CSV backup
Current snapshot written to backup_data_server.csv. File accumulates rows every 10 minutes until the process restarts.
T + 3600 s — First device ping
All three Pico devices pinged; online/offline status logged to console.
T + 7200 s — First Discord RAM report
RAM usage summary for Pico 1, 2, and 3 posted to the Discord status channel.
Ongoing — Continuous reception
data_recv() loops indefinitely, accepting and processing each incoming TCP connection from any Pico device.

How to Display Data from the Picos

Three methods to get sensor readings out of the server and onto a screen — from the Pico itself, to the cloud, to a custom live dashboard.

1
MicroPython — Send data from each Pico

Each Pico opens a TCP socket to the server on port 9991 and sends readings as plain key-value lines. Copy the template below onto each Pico, changing the PICO_ID (1, 2, or 3) and the SERVER_IP.

import socket, time
from dht import DHT22
from machine import Pin
import gc

SERVER_IP  = '192.168.1.X'   # ← replace with your server IP
SERVER_PORT = 9991
PICO_ID    = 1               # ← 1, 2, or 3
INTERVAL   = 30              # seconds between sends

sensor = DHT22(Pin(15))
door_pin = Pin(16, Pin.IN)
pump_pin = Pin(17, Pin.IN)

def send_data():
    sensor.measure()
    ram_free = gc.mem_free()
    ram_total = gc.mem_alloc() + ram_free
    ram_pct = round((1 - ram_free / ram_total) * 100, 1)

    payload = (
        f"HM{PICO_ID}:{sensor.humidity()}\n"
        f"TP{PICO_ID}:{sensor.temperature()}\n"
        f"DRRS{PICO_ID}:{door_pin.value()}\n"
        f"PS{PICO_ID}:{pump_pin.value()}\n"
        f"RAM{PICO_ID}:{ram_pct}\n"
    )
    try:
        s = socket.socket()
        s.connect((SERVER_IP, SERVER_PORT))
        s.send(payload.encode())
        s.close()
    except Exception as e:
        print(f"Send error: {e}")

while True:
    send_data()
    time.sleep(INTERVAL)
# Same as Pico 1 — change PICO_ID only
PICO_ID = 2
SERVER_IP = '192.168.1.X'

# All prefix labels become HM2, TP2, DRRS2, PS2, RAM2 automatically
# Same as Pico 1 — change PICO_ID only
PICO_ID = 3
SERVER_IP = '192.168.1.X'

# All prefix labels become HM3, TP3, DRRS3, PS3, RAM3 automatically
Pin mapping: DHT22 data → GP15 · Door sensor → GP16 · Pump relay → GP17. Adjust Pin() numbers to match your wiring.

To fetch the latest value programmatically:

# Python — read latest humidity from Pico 1 channel
import requests
CH_ID  = 'YOUR_CHANNEL_ID'
RD_KEY = 'YOUR_READ_API_KEY'
r = requests.get(
    f"https://api.thingspeak.com/channels/{CH_ID}/fields/1/last.json",
    params={"api_key": RD_KEY}
)
print(f"Latest humidity: {r.json()['field1']} %")
3
Read data directly from the CSV backup

The server writes backup_data_server.csv every 10 minutes. You can load it with pandas, Excel, or any spreadsheet tool.

import pandas as pd

df = pd.read_csv('backup_data_server.csv', parse_dates=['Time'])

# Latest reading per greenhouse
latest = df.iloc[-1]
print(f"Semi-Closed  — Humidity: {latest['Humidity SC']}%  Temp: {latest['TP SC']}°C")
print(f"Fully-Closed — Humidity: {latest['HM C']}%  Temp: {latest['TP C']}°C")
print(f"Free Planting — Humidity: {latest['HM FREE']}%  Temp: {latest['TP FREE']}°C")

# Plot humidity over time
import matplotlib.pyplot as plt
df.plot(x='Time', y=['Humidity SC', 'HM C', 'HM FREE'])
plt.title('Greenhouse Humidity Over Time')
plt.ylabel('Humidity (%)')
plt.show()
