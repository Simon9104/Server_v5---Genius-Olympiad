TCP Data Protocol

Each Pico connects to 0.0.0.0:9991 and sends plain-text key-value pairs (one per line). The server parses every line and updates the corresponding global variable.

TP1: TP2: TP3: Temperature Float, °C TP2:22.3 All 3 Picos

Async Task Architecture

The server runs 7 concurrent asyncio tasks inside a single Python process.

data_recv()
⏱ Continuous (0.2 s delay)
TCP socket listener. Accepts incoming connections from any Pico on port 9991, decodes the payload, parses each line, and updates global sensor variables. Increments a per-sensor sequence counter on every successful parse.
data_send()
⏱ Every 120 seconds
Sends current sensor values to ThingSpeak via HTTPS GET for all four channels (3 greenhouse channels + 1 RAM channel). On failure, posts an error alert to the Discord error channel.
transfer_discord()
⏱ Every 7200 seconds (2 hr)
Posts a RAM usage summary for all three Pico devices to the main Discord status channel. Allows remote monitoring of device health without SSH access.
backup_data()
⏱ Every 600 seconds (10 min)
Appends the current sensor snapshot (timestamp + all 12 readings) to an in-memory list and writes the entire list to backup_data_server.csv using pandas.
control_RPI()
⏱ Every 3600 seconds (1 hr)
Pings Pico 1 at 192.168.1.59 using the system ping command. Prints ONLINE / OFFLINE to the console. Helps detect network outages early.
control_RPI2() & control_RPI3()
⏱ Every 3600 seconds (1 hr)
Same as control_RPI() but for Pico 2 and Pico 3 at 192.168.1.118. Each runs as an independent task so one slow ping doesn't block the others.
Operational Timeline
T + 0 s — Startup
Server binds TCP :9991, initialises all globals to 0, posts "Program was started" to Discord main channel.
T + 120 s — First ThingSpeak push
All four ThingSpeak channels updated with the first batch of sensor data. Heartbeat "program is running!" printed to console.
T + 600 s — First CSV backup
Current snapshot written to backup_data_server.csv. File accumulates rows every 10 minutes until the process restarts.
T + 3600 s — First device ping
All three Pico devices pinged; online/offline status logged to console.
T + 7200 s — First Discord RAM report
RAM usage summary for Pico 1, 2, and 3 posted to the Discord status channel.
Ongoing — Continuous reception
data_recv() loops indefinitely, accepting and processing each incoming TCP connection from any Pico device.
